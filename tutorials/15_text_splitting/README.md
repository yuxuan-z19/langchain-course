# 第15节：文档切分技术

## 概述

文档切分（Text Splitting）是RAG（检索增强生成）系统中的核心环节，直接影响检索质量和最终生成效果。合适的切分策略能够确保每个文档片段包含完整、相关的信息，提高检索的准确性和生成的质量。

## 为什么文档切分是RAG的重中之重？

### 1. 检索精度影响
- **粒度过大**：包含过多无关信息，降低检索精度
- **粒度过小**：信息不完整，缺乏上下文
- **合适粒度**：平衡信息完整性和检索精度

### 2. 模型处理限制
- 大多数嵌入模型有输入长度限制
- 过长文本可能导致信息丢失
- 合适长度确保嵌入质量

### 3. 生成质量保证
- 相关性高的片段提供更好的上下文
- 避免信息冗余和噪声
- 提高回答的准确性和相关性

## 文档切分方法分类

### 1. 基于长度的切分

#### 字符数切分
- **原理**：按固定字符数切分文档
- **优点**：简单直接，计算效率高
- **缺点**：可能破坏语义完整性
- **适用场景**：结构化文档，预处理阶段

#### Token数切分
- **原理**：按模型token数量切分
- **优点**：符合模型输入限制
- **缺点**：需要tokenizer，计算开销大
- **适用场景**：需要精确控制模型输入长度

### 2. 基于文本结构的切分

#### 段落切分
- **原理**：按自然段落边界切分
- **优点**：保持语义完整性
- **缺点**：段落长度不均匀
- **适用场景**：文章、报告等结构化文本

#### 句子切分
- **原理**：按句子边界切分，可组合多个句子
- **优点**：语义边界清晰
- **缺点**：可能过于细粒度
- **适用场景**：需要精细控制的场景

#### 递归切分
- **原理**：按优先级顺序尝试不同分隔符
- **优点**：智能选择最佳切分点
- **缺点**：逻辑相对复杂
- **适用场景**：通用文本处理

### 3. 基于文档格式的切分

#### Markdown标题切分
- **原理**：按Markdown标题层级切分
- **优点**：保持文档结构和层次
- **缺点**：仅适用于Markdown格式
- **适用场景**：技术文档、Wiki、博客

#### 代码切分
- **原理**：按代码结构（函数、类、模块）切分
- **优点**：保持代码逻辑完整性
- **缺点**：需要语言特定的解析器
- **适用场景**：代码文档、API文档

#### HTML/XML切分
- **原理**：按HTML标签或XML元素切分
- **优点**：保持文档结构
- **缺点**：需要解析器，处理复杂
- **适用场景**：网页内容、结构化文档

### 4. 基于语义的切分

#### 语义相似度切分
- **原理**：计算文本片段间的语义相似度
- **优点**：保持语义连贯性
- **缺点**：计算复杂，需要嵌入模型
- **适用场景**：高质量要求的RAG系统

#### 主题分割
- **原理**：识别文本中的主题变化点
- **优点**：按主题组织信息
- **缺点**：需要主题模型，计算开销大
- **适用场景**：长文档、多主题内容

## 切分粒度对RAG效果的影响

### 1. 检索阶段影响

| 粒度 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| 细粒度 | 检索精确，噪声少 | 上下文不足，信息碎片化 | 事实查询，简单问答 |
| 中粒度 | 平衡精确性和完整性 | 需要精心调优 | 通用RAG应用 |
| 粗粒度 | 上下文丰富，信息完整 | 噪声多，检索不精确 | 复杂推理，长文本生成 |

### 2. 生成阶段影响

- **上下文窗口利用**：合适粒度最大化有效信息密度
- **推理能力**：足够上下文支持复杂推理
- **一致性**：避免信息冲突和矛盾

## 最佳实践和选择建议

### 1. 选择原则

1. **文档类型优先**：根据文档结构选择切分方法
2. **应用场景适配**：考虑检索和生成的具体需求
3. **性能平衡**：在质量和效率间找到平衡点
4. **迭代优化**：通过实验不断调优参数

### 2. 组合策略

- **层次化切分**：先粗后细，多层次处理
- **混合方法**：结合多种切分策略
- **动态调整**：根据内容特点动态选择方法

### 3. 质量评估

- **覆盖率**：重要信息的保留程度
- **相关性**：片段与查询的匹配度
- **完整性**：语义和逻辑的完整程度
- **一致性**：片段间信息的一致性

### 4. 参数调优

- **chunk_size**：根据模型和应用调整
- **chunk_overlap**：保证信息连续性
- **分隔符优先级**：根据文档特点设置
- **最小片段长度**：避免过短无意义片段

## 总结

文档切分是RAG系统成功的关键因素。选择合适的切分策略需要综合考虑文档特点、应用需求和性能要求。通过理解不同切分方法的原理和特点，结合实际测试和评估，您可以为您的RAG系统构建最优的文档处理流程。
